# activator

## ideas

1. 很多ai的订阅套餐都会有流量限制，一般为每a个小时限制小于等于b个请求次数，这里的a个小时并不是固定的时间，而是当用户发送了一个请求后激活，之后的a个小时内只能发送b个请求。
2. 但是很多用户使用ai时可能很稀疏，即隔很长一段时间都没有使用，但是一旦使用就会消耗大量的请求次数——甚至超过每a小时的限制。
3. 为此，我们可以考虑设计一个activator每隔a个小时发送一个激活用的请求，这样的话，如果用户在一次请求对应的a小时的末尾开始使用，理论上来说，它的短时间可以使用的请求次数是翻倍的。

## userstory

1. 这是一个在终端中使用的工具，用户在终端中配置以下参数

- a 请求发送间隔
- b 单词请求的token数
- url
- apikey
- model
- 接口类型

1. 回车后，程序会每隔a时间像对应的url，apikey，model发送一个请求，（请求使用 'the' 填充，共有b个'the' ）。
2. 同时终端中也应该返回日志，包含上次请求是否成功，服务器的返回，以及下次请求的时间。
3. 支持各种形式的接口 如openai-compatible和anthropic.
